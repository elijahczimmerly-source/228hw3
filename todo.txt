🚨 CRITICAL BUG WARNING: Your Math
  I noticed in your remove(int pos) method, you are doing this:
  int nodePos = pos / nodeSize; and int offset = pos % nodeSize;

  This will not work for a Stout List.

  In a standard ArrayList, elements are perfectly packed. But in a StoutList, nodes can be half-full. For example, if $M=4$:
   * Node 0 might have 2 elements: [A, B, -, -]
   * Node 1 might have 3 elements: [C, D, E, -]

  If you want logical index 3 (which is 'D'), your math 3 / 4 would say it's in Node 0. But Node 0 only has 2 things! It's actually in Node 1.

  How to fix your find logic:
  You must iterate through the nodes and "count" as you go:
   1. Start at head.next.
   2. If pos is less than current.count, your target is in this node at offset = pos.
   3. Otherwise, subtract current.count from pos and move to the next node (pos -= current.count; current = current.next).
   4. Repeat until you find it.

  Example for Index 3:
   1. Node 0 has count = 2. Is 3 < 2? No.
   2. pos = 3 - 2 = 1. Move to Node 1.
   3. Node 1 has count = 3. Is 1 < 3? Yes!
   4. Your answer: Node 1, Offset 1.
